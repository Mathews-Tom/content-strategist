# Channel Specifications

Format constraints, structural patterns, and optimization rules per distribution channel.

---

## LinkedIn

| Parameter | Specification |
|-----------|--------------|
| Character limit | 3,000 characters (post body) |
| Ideal length | 1,200-1,800 characters for engagement |
| Hook window | First 2 lines visible before "see more" (~210 chars) |
| Hashtags | 3-5 relevant, placed at end |
| Mentions | Tag people/companies in first comment, not body |
| Media | Single image or carousel outperforms text-only |
| CTA placement | Final line, direct and specific |

### Hook Patterns

- **Contrarian opener**: "Most teams get [topic] wrong. Here's why."
- **Data lead**: "[Specific number] — that's how many [metric]."
- **Story entry**: "Last quarter, we [specific event]."
- **Question**: "What if [assumption] is the reason [problem]?"

### Structure

1. Hook (1-2 lines)
2. Context or story (2-3 lines)
3. Key insight with evidence (3-5 lines)
4. Takeaway or framework (2-4 lines)
5. CTA or question (1 line)

## Blog / Long-Form Article

| Parameter | Specification |
|-----------|--------------|
| Standard length | 800-1,500 words |
| Deep dive length | 1,500-3,000 words |
| Subheading frequency | Every 200-300 words |
| Paragraph length | 2-4 sentences max |
| Code blocks | Syntax-highlighted, with context line above |
| Images/diagrams | 1 per 500 words minimum for technical content |

### Structure

1. Title (action-oriented, specific)
2. Opening paragraph (problem statement, why the reader cares)
3. Context section (background, current state)
4. Body sections (3-5 key points, each with evidence)
5. Conclusion (summary, actionable takeaway)
6. CTA (next article, newsletter, product)

### SEO Checklist

- Primary keyword in title, H1, first paragraph, one H2
- Meta description: 150-160 characters
- Internal links: 2-3 to related content
- External links: 1-2 to authoritative sources

## Slide Deck / Presentation

| Parameter | Specification |
|-----------|--------------|
| Slide count | 10-20 slides for standard presentations |
| Words per slide | 20-30 maximum |
| Title slides | One main idea, 6 words or fewer |
| Bullet points | 3-5 per slide, single line each |
| Font size | 24pt minimum for body, 36pt for titles |
| Speaker notes | 50-100 words per slide |

### Narrative Arc

1. Title + speaker intro (1 slide)
2. Problem / context (1-2 slides)
3. Key points with evidence (3-5 slides, one per point)
4. Demo or example (1-2 slides)
5. Summary (1 slide)
6. CTA / next steps (1 slide)

## PDF Report / Document

| Parameter | Specification |
|-----------|--------------|
| Cover page | Title, author, date, version, logo placeholder |
| TOC | Required for documents over 3 pages |
| Executive summary | Required for documents over 5 pages |
| Section numbering | Hierarchical (1, 1.1, 1.1.1) |
| Page numbers | Bottom center or bottom right |
| Headers/footers | Document title in header, page number in footer |

### Section Order

1. Cover page
2. Table of contents
3. Executive summary
4. Body sections
5. Appendices
6. References

---

## Cross-Channel Rules

- Core message appears in every format — adapted, not duplicated
- Data points are consistent across channels (no contradictory numbers)
- Each channel takes a distinct angle on the same topic
- No verbatim duplication between channels
- Tone shifts per channel but voice remains consistent
